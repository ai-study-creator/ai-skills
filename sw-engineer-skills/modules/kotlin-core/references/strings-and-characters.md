# Strings and Characters

## Characters
- Understand supplementary characters and escape sequences.
- Be careful with character arithmetic and comparison when Unicode matters.
- Distinguish code units from user-visible characters.

## Strings
- Prefer the string literal form that is easiest to read.
- Use multiline strings when they improve structured text.
- Keep escaping rules obvious in public or machine-facing text.

## Common pitfalls
- Assuming one visual character equals one `Char`
- Mixing display text rules with protocol or storage encoding rules
