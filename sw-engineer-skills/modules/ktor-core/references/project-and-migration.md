# Project Setup and Migration

## Project setup
- Keep dependencies, engines, and plugins aligned with the intended deployment model.
- Use the project generator or CLI when it reduces setup drift.
- Make the build toolchain explicit.

## Migration
- Read migration notes before upgrading Ktor major versions.
- Check server and client API changes separately.
- Expect plugin and module organization to change across major upgrades.

## Practical note
- Project setup is part of the application architecture, not just scaffolding.
