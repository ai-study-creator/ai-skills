# Coroutines Index

Use these files when the task involves coroutine structure, cancellation, dispatchers, channels, flows, debugging, or async design.

## Topic map
- `coroutines-basics.md` - suspend functions, builders, scope, context, runBlocking
- `coroutines-structured-concurrency.md` - parent/child jobs, lifecycle, cancellation propagation, scopes
- `coroutines-dispatchers.md` - Default, IO, custom dispatchers, blocking boundaries
- `coroutines-channels-flows.md` - channels, flow, shared flow, state flow, backpressure-style reasoning
- `coroutines-debugging.md` - IntelliJ debugger, coroutine frames, flow debugging, diagnostics
- `coroutines-best-practices.md` - API design, testing, shared state, performance, common pitfalls

## Reading order
1. Read `coroutines-basics.md` for language and library fundamentals.
2. Read `coroutines-structured-concurrency.md` for lifecycle and cancellation.
3. Read `coroutines-dispatchers.md` for thread and blocking policy.
4. Read `coroutines-channels-flows.md` when coroutines exchange data.
5. Read `coroutines-debugging.md` when tracing or diagnosing behavior.
6. Read `coroutines-best-practices.md` for project-level decisions.
