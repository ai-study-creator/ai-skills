# Kotlin Reference Index

Use these files when the task involves Kotlin source, coroutines, flows, JVM interop, or Kotlin-specific performance work.

## Topic map
- `getting-started.md` - the language overview, project setup, and first steps
- `syntax-types.md` - basics, type system, data classes, sealed types, inline/value classes
- `strings-and-characters.md` - string literals, escapes, supplementary characters, character operations
- `functions-and-classes.md` - functions, lambdas, constructors, properties, member functions, delegation
- `null-safety.md` - nullable types, smart casts, contracts, safe handling
- `coroutines-index.md` - coroutine topics, structured concurrency, dispatchers, channels, flow, debugging
- `flow-streaming.md` - flows, backpressure-like behavior, streaming pipelines
- `jvm-interoperability.md` - Java interop, annotations, overloads, SAMs, platform types
- `build-and-tooling.md` - Gradle, Maven, compiler options, caches, kapt, plugins
- `multiplatform-notes.md` - when cross-platform concerns matter
- `performance-idioms.md` - idiomatic Kotlin that still performs well
- `jvm-and-library-compatibility.md` - JVM bytecode, metadata, records, plugins, interop, compatibility shifts
- `compatibility-and-migrations.md` - deprecations, migration guides, and compatibility notes

## Reading order
1. Read `getting-started.md` and `syntax-types.md` for language fundamentals.
2. Read `functions-and-classes.md` and `strings-and-characters.md` for core Kotlin implementation work.
3. Read `null-safety.md` when the task touches boundary validation or defensive code.
4. Read `coroutines-index.md` and `flow-streaming.md` for async and streaming work.
5. Read `jvm-interoperability.md` and `build-and-tooling.md` when Kotlin touches Java, frameworks, or the build toolchain.
6. Read `performance-idioms.md`, `jvm-and-library-compatibility.md`, and `compatibility-and-migrations.md` when optimizing or upgrading.
